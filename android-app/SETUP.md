# 📱 안드로이드 알림 수집 앱 셋업 가이드

## 개요
이 앱은 카드 결제 알림(푸시/SMS)을 자동으로 감지하여 Google Apps Script 웹훅으로 전송합니다.
앱을 한 번 설정하면 이후에는 백그라운드에서 자동으로 동작합니다.

---

## 설치 방법

### 방법 1: APK 직접 설치
1. 프로젝트를 빌드합니다:
   ```bash
   cd android-app
   set JAVA_HOME=jdk17\jdk-17.0.13+11
   gradlew.bat assembleDebug
   ```
2. 생성된 APK 파일 위치:
   `android-app/app/build/outputs/apk/debug/app-debug.apk`
3. APK 파일을 안드로이드 기기에 전송하여 설치합니다.

### 방법 2: Android Studio에서 실행
1. Android Studio에서 `android-app` 폴더를 엽니다.
2. USB로 기기를 연결하고 ▶ Run 버튼을 클릭합니다.

---

## 초기 설정 (앱 내)

### 1. 알림 접근 권한 허용
앱을 처음 열면 상단에 상태 카드가 표시됩니다:
- **🔴 비활성** → "권한 설정하러 가기" 버튼 클릭
- 시스템 설정에서 **SmartBudget** 앱의 알림 접근 권한을 **ON**
- 앱으로 돌아오면 **🟢 수집 중** 상태로 변경

### 2. 웹훅 URL 입력
- [Apps Script 셋업](file:///c:/Users/wodls/Desktop/안티그래비티/가계부 어플/apps-script/SETUP.md)에서 배포한 URL을 입력
- "URL 저장" 버튼 클릭
- 형식: `https://script.google.com/macros/s/AKfycb.../exec`

### 3. 앱 종료 후 확인
- 이 앱은 **홈 버튼을 눌러 닫아도** 백그라운드에서 계속 실행됩니다.
- 기기를 재부팅해도 자동으로 시작됩니다.

## 배터리 최적화 비활성화 (권장)
일부 안드로이드 기기에서는 절전 모드가 알림 감지를 방해할 수 있습니다.

1. **설정 → 앱 → SmartBudget → 배터리**
2. **"배터리 사용 제한 없음"** 또는 **"제한 없음"** 선택
3. (삼성 기기) **설정 → 배터리 → 백그라운드 사용 제한** 에서 SmartBudget 제외

## 문제 해결

**Q: 알림이 감지되지 않아요**
A: 다음을 확인하세요:
1. 알림 접근 권한이 허용되어 있는지
2. 배터리 최적화가 비활성화되어 있는지
3. 카드사 앱에서 "결제 알림" 설정이 켜져 있는지

**Q: 스프레드시트에 기록되지 않아요**
A: 웹훅 URL이 올바르게 입력되어 있는지 확인하세요. Apps Script가 정상 배포되어 있는지도 확인하세요.

**Q: 카테고리가 "기타"로만 분류돼요**
A: Apps Script의 Gemini API 키가 올바르게 설정되어 있는지 확인하세요.
